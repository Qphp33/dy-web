# 位置信息模块抖音风 UI 设计

## 目标

把位置信息模块从偏后台表格风，优化成更清爽的浅色抖音后台管理界面：保留项目原有白蓝配色，减少额外头部装饰，重点优化按钮、表格、状态胶囊和内容卡片的层级，同时不改变现有接口、路由和业务逻辑。

## 范围

- 列表页：`src/views/module/Address/index.vue`
- 详情页：`src/views/module/Address/detailPage.vue`
- 不调整接口请求、分页、弹窗、删除、编辑、新建任务等业务行为。
- 不接入新的第三方 UI 库。

## 视觉方向

- 背景：沿用项目现有浅蓝到白色渐变底，不额外增加大面积彩色光斑。
- 主色：沿用蓝色 `#1677ff` / `#1890ff`，少量使用抖音红 `#fe2c55` 和青色 `#25d6d2` 做状态和视觉记忆点。
- 容器：白色浅色卡片、柔和阴影和圆角，避免过重的玻璃态装饰。
- 交互：表格行 hover 不改变背景色，只保留可点击指针；卡片 hover 保持轻微即可。
- 信息层级：标题、任务名、昵称使用深蓝黑；说明、时间、备注使用灰蓝色。

## 页面设计

### 列表页

列表页顶部不再放独立标题 card，只保留操作区：

- 右侧：新建按钮和表格设置/刷新按钮。
- 表格：保留 Ant Design Vue 表格能力，改为更轻、更圆润的玻璃态浅色表格皮肤。
- 标签、状态、设备 ID：使用胶囊样式，减少后台系统的生硬感。

### 详情页

保留现有结构，重塑视觉：

- 地区卡片保持左图右评论，使用项目原有浅色白蓝内容卡。
- 排序筛选栏保持白底吸顶工具条，按钮与选择器使用圆角胶囊。
- 用户视频卡片保持浅色信息流卡片：头像更突出，属地和 ID 使用浅蓝/浅青胶囊，主要动作仍用蓝色。
- 空状态、分页、弹窗入口保持可用。

## 验证

- 运行 `npm run build`，确保 Vue CLI 生产构建通过。
- 检查 Git diff 只涉及设计文档、计划文档和位置信息相关页面。
- 生成本地 Git commit，远端 GitHub 推送仅在本机凭据可用时执行；不强推、不覆盖远端 `main`。
