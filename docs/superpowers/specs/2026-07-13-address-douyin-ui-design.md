# 位置信息模块抖音风 UI 设计

## 目标

把位置信息模块从偏后台表格风，优化成更接近抖音数据内容后台的视觉：深色沉浸背景、高对比卡片、玫红与青色撞色、信息流式内容卡片，同时不改变现有接口、路由和业务逻辑。

## 范围

- 列表页：`src/views/module/Address/index.vue`
- 详情页：`src/views/module/Address/detailPage.vue`
- 不调整接口请求、分页、弹窗、删除、编辑、新建任务等业务行为。
- 不接入新的第三方 UI 库。

## 视觉方向

- 背景：近黑渐变，叠加玫红和青色柔光，营造短视频平台感。
- 主色：`#fe2c55` 作为行动色，`#25f4ee` 作为科技感点缀。
- 容器：暗色半透明卡片，圆角更大，边框带轻微高光。
- 交互：行 hover 和卡片 hover 使用轻微上浮、霓虹描边和阴影。
- 信息层级：标题、任务名、昵称使用高亮白色；说明、时间、备注使用低饱和灰色。

## 页面设计

### 列表页

顶部由普通表格标题改成“位置信息采集”任务中心：

- 左侧：模块名、说明文案、任务总数、设备总数。
- 右侧：新建按钮和表格设置按钮。
- 表格：保留 Ant Design Vue 表格能力，改为暗色表格皮肤。
- 标签、状态、设备 ID：使用胶囊样式，减少后台系统的生硬感。

### 详情页

保留现有结构，重塑视觉：

- 地区卡片保持左图右评论，但改成深色内容卡。
- 排序筛选栏改成暗色吸顶工具条。
- 用户视频卡片改成抖音信息流卡片：头像更突出，状态按钮使用玫红，属地和 ID 使用青色/暗色胶囊。
- 空状态、分页、弹窗入口保持可用。
